Big O Notation

  • A mathematical notation that shows how efficient an algorithm is in the worst-case scenario relative to its input size.
  • It captures how much time or space an algorithm would require approximately as the input size grows: time complexity & space complexity
  • f(n) = O(inputSize)

What is Time Complexity?

  • Does NOT represent the actual time required to execute a code
  • The computational time (CPU time) it will take to execute an algorithm, determined by the input size and the solution of the algorithm.

Selected Types of Time Complexity

Constant time: O(1)

  • Not dependent on the input size n
  • The runtime is the same irrespective of the input size
  • e.g. finding the first element of an array/stack

Logarithmic time: O(log n)

  • When the size of the input is reduced in each step
  • The number of operations is not the same as the input
  • The execution time increases, but at a decreasing rate
  • e.g. Binary trees or binary search functions
  • Divide and conquer approach
    1. Divide the given problem into sub-problems of the same type.
    2. Recursively solve these sub-problems.
    3. Appropriately combine the sub-answers to answer the given problem.

Linear time: O(n)

  • The runtime increases in direct propertion with the input size
  • i.e. processes the input in n number of operations
  • Looking at every item in a list to complete a task
    • e.g. using a for loop to check the values in the input data

Quadratic: O(n^2)

  • A non–linear time complexity
  • The running time increases non-linearly (n^2) with the length of the input
  • Execution time increases at an increasing rate
  • e.g. nested loops: O(n) * O(n) = O(n^2)

Exponential: O(c^n)

  • Execution time increases exponentially as input n grows
  • The runtime doubles with each addition of the input n, often iterating through all subsets of the input elements
  • Used when we don’t know the best solution so we have to try every possible combination or permutation on the data
  • Brute-Force algorithms that blindly iterate the possible solutions in search of one or more solutions that satisfy a condition
  • Used in brute-force attach: also known as an exhaustive search, is a cryptographic hack that relies on guessing possible combinations of a targeted password until the correct password is discovered.

How to Calculate Time Complexity

  • Generic steps to determine the time complexity
    1. List all the basic operations
    2. Count the number of times each gets executed
    3. Sum all the counts to get an equation in terms of n

Example Calculation

var main = function() {
  var sum = 0;
  for (var i = 0; i < 5; i++) {
    sum = sum + i;
  }
  return 0;
}
Operations Number of Execution
var sum = 0; 1
for (var i = 0; i < 5; i++) 6
sum = sum + i; 5
return 0; 1

Calculation:

  • 1 + 6 + 5 + 1

Rewriting this with the input n:

  • 1 + (n + 1) + n + 1
  • = 2n + 3

To find Big-O notation:

  • Discard the leading constants: 2n+3 => n+3
  • Ignore the lower order terms: n+3 => n => O(n)
